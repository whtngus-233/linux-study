## 1. 학습 개요
- 환경: Ubuntu 24.04 (VMware), PuTTY(SSH)
- 범위: 사용자 관리, 권한, 웹/FTP 서버 구축
- 목표: 단순 명령어 암기가 아닌 실제 서버 운영 경험 확보

---

## 2. 트러블슈팅 (핵심)

### 🔹 403 Forbidden 발생
- 문제: 웹 페이지 접근 불가
- 원인: index.html 권한 600
- 해결: chmod 644 index.html
- 결과: 정상 출력

👉 포인트: 웹 서버는 왜 other 권한이 필요한가?

---

### 🔹 FTP 업로드 실패
- 문제: Permission denied
- 원인: vsftpd write_enable 비활성화
- 해결: 설정 변경 후 재시작
- 결과: 업로드 성공

👉 포인트: 설정 문제 vs 권한 문제 구분 방법

---

### 🔹 ping 통신 불가
- 문제: 응답 없음
- 원인: 방화벽 ICMP 차단
- 해결: ICMP 허용
- 결과: 통신 정상

👉 포인트: 네트워크 vs 서버 문제 구분

---

## 3. 실습 결과

- Apache 웹 서버 구축 및 페이지 배포 완료
- FTP 서버 통해 외부 파일 업로드 성공
- 권한 변경에 따른 서비스 동작 변화 확인

---

## 4. 핵심 개념 정리

- chmod 755 → 디렉터리 접근 가능
- chmod 644 → 웹 서비스 필수 권한
- 디렉터리는 x 없으면 접근 자체 불가
- 권한 문제는 서비스 장애로 직결됨

---

## 5. 명령어 정리

| 명령어 | 설명 | 예제 |
|--------|------|------|
| pwd | 현재 위치 | pwd |
| ls | 목록 확인 | ls -l |
| chmod | 권한 변경 | chmod 755 file |
| chown | 소유자 변경 | chown user:file |
| useradd | 사용자 생성 | useradd -m user1 |
